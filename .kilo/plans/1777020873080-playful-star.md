# 互动体验页面布局升级计划
## 任务目标
1. 修复功能窗口透明度问题，提升内容可见度
2. 替换浮窗形式，改用二分法分屏布局
3. 实现功能交互：点击某一功能，中部展开详情，另一功能略微缩小
4. 全景浏览集成到布局内，取消弹窗形式

## 实施步骤
### 1. 布局重构（核心修改）
- 移除原有的功能卡片网格布局
- 改为左右并排的双功能卡片布局，各占50%宽度
- 移除功能内容区域的独立卡片，将内容直接集成到选中的功能卡片内
- 移除全景浏览弹窗，将全景展示区域集成到选中的卡片中

### 2. 交互逻辑优化
- 点击左侧功能卡片时：
  - 左侧卡片宽度扩展为75%，高度自动适配内容
  - 右侧卡片宽度压缩为25%，仅保留图标和标题
  - 左侧卡片内展开对应功能的完整内容（知识测验/全景浏览）
- 点击右侧功能卡片时：
  - 右侧卡片宽度扩展为75%，高度自动适配内容
  - 左侧卡片宽度压缩为25%，仅保留图标和标题
  - 右侧卡片内展开对应功能的完整内容

### 3. 样式调整
- 提升功能卡片背景不透明度，从0.7改为0.95，增强可见度
- 调整阴影和边框，提升层次感
- 保留现有动画效果（墨韵转场、卡片动效、测验动效等）
- 响应式适配：移动端保持上下堆叠布局

### 4. 全景浏览集成
- 移除原有的fixed全景弹窗
- 将全景浏览区域直接嵌入到展开的功能卡片内
- 保留惯性滑动、缩放等原有全景功能
- 新增关闭按钮，点击后回到卡片初始状态

## 影响范围
仅修改`src/views/Interactive.vue`文件，其他功能不受影响

## 实现细节说明
1. 保留所有现有功能逻辑，仅调整布局结构
2. 复用现有动画效果，保证体验一致性
3. 遵循现有古风UI设计规范，使用统一的主题色和样式类
4. 保证移动端适配，小屏幕下自动切换为垂直堆叠布局
