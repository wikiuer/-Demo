# 古建筑知识平台AI功能升级计划
## 项目概述
当前AI功能为一次性返回响应，用户等待时间长，缺少知识库匹配、页面上下文感知和答题模块联动功能。本次升级将实现四个核心功能，提升用户体验。
## 功能实现计划
### 1. 流式输出响应实现
- 修改API调用方式，使用DeepSeek SSE流式接口
- 实现逐字输出效果，类似主流Chat产品体验
- 适配现有对话UI，确保流式输出显示正常
### 2. 知识库匹配功能实现
- 将public/data/buildings.json的所有建筑信息整合到System Prompt
- 实现建筑名称识别功能，当用户提到收录的建筑时，自动在回答末尾添加跳转链接
- 优化回答逻辑，确保回答优先使用知识库内容，避免信息不一致
### 3. 页面上下文感知实现
- 监听路由变化，识别当前是否在建筑详情页
- 在建筑详情页打开AI时，自动显示提示语："你正在浏览[建筑名]，有任何相关问题都可以问我"
- 自动将当前建筑的完整信息作为上下文传入AI请求，提升回答针对性
### 4. 答题模块联动实现
- 在QuizFloatButton组件中添加答错触发AI讲解的逻辑
- 用户答错题目时，自动打开AI聊天窗口并传入对应题目知识点，AI给出讲解
- 实现AI根据用户错题记录针对性出题的功能
## 实施步骤
1. 第一阶段：实现流式输出响应
2. 第二阶段：实现知识库匹配功能
3. 第三阶段：实现页面上下文感知功能
4. 第四阶段：实现答题模块联动功能
5. 测试与优化：整体功能测试，修复bug，优化体验
## 预计修改文件
- src/components/AIChat.vue：核心AI组件修改，包含流式输出、上下文感知、知识库匹配
- src/components/QuizFloatButton.vue：答题组件修改，添加联动逻辑
- 可能需要新增工具函数处理建筑信息匹配
